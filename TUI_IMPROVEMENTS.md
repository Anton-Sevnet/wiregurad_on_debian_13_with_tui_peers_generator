# Улучшения TUI интерфейса

## Проблема

При использовании `dialog` в некоторых терминалах или при неправильной кодировке вместо красивых рамок отображались символы "q", "l", "k", "x", "m" и другие артефакты.

## Решение

В скрипт добавлены следующие улучшения:

### 1. Настройка кодировки и терминала

В начале скрипта добавлены настройки для корректного отображения:

```bash
export LANG=C.UTF-8
export LC_ALL=C.UTF-8
export TERM=xterm-256color
```

### 2. Использование ASCII символов

Все вызовы `dialog` теперь используют опцию `--ascii-lines`, которая заменяет специальные символы Unicode на простые ASCII символы (+, -, |):

```bash
dialog --ascii-lines --no-shadow --title "Заголовок" ...
```

### 3. Упрощение интерфейса

Добавлена опция `--no-shadow` для удаления теней, что делает интерфейс более чистым и простым.

### 4. Очистка экрана

Добавлена опция `--clear` перед каждым диалогом для очистки экрана и более аккуратного отображения.

### 5. Заголовок приложения

Добавлен `--backtitle "WireGuard Peer Manager"` для отображения названия приложения в верхней части экрана.

## Результат

Теперь интерфейс отображается с:
- ✅ Чистыми ASCII рамками (+, -, |)
- ✅ Без теней
- ✅ С очисткой экрана между диалогами
- ✅ С заголовком приложения
- ✅ Корректной кодировкой

После применения изменений интерфейс будет выглядеть так:

```
+------------------------------------------+

+------------------------------------------+

|                                          |
```

## Дополнительные настройки

Если проблемы с отображением сохраняются, можно попробовать:

1. **Установить правильную кодировку терминала:**
   ```bash
   export LANG=en_US.UTF-8
   ```

2. **Использовать другой терминал:**
   - xterm
   - gnome-terminal
   - konsole

3. **Проверить настройки терминала:**
   ```bash
   echo $TERM
   ```

4. **Использовать whiptail вместо dialog:**
   Скрипт автоматически использует whiptail, если dialog недоступен или работает некорректно.
