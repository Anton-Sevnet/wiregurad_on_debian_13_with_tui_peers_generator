# Принудительное обновление репозитория на продакшене

## Проблема
При `git pull` возникает конфликт из-за локальных изменений в файлах.

## Решение 1: Принудительный сброс (рекомендуется для продакшена)

```bash
# Сбросить все локальные изменения и синхронизировать с удаленным репозиторием
git fetch origin
git reset --hard origin/main
```

**Внимание:** Это удалит все локальные изменения без возможности восстановления!

## Решение 2: Сохранение локальных изменений

Если нужно сохранить локальные изменения:

```bash
# Сохранить изменения во временное хранилище
git stash

# Обновить репозиторий
git pull

# Применить сохраненные изменения обратно (если нужно)
git stash pop
```

## Решение 3: Отмена изменений в конкретном файле

Если нужно обновить только конкретный файл:

```bash
# Отменить изменения в файле
git checkout -- wg-add-peer.sh

# Обновить репозиторий
git pull
```

## Решение 4: Полная переустановка (крайний случай)

Если ничего не помогает:

```bash
# Удалить локальный репозиторий
cd ..
rm -rf wiregurad_on_debian_13_with_tui_peers_generator

# Клонировать заново
git clone https://github.com/Anton-Sevnet/wiregurad_on_debian_13_with_tui_peers_generator.git
cd wiregurad_on_debian_13_with_tui_peers_generator
```

## Для вашего случая (продакшен)

Выполните:

```bash
git fetch origin
git reset --hard origin/main
```

Это принудительно обновит локальную версию до последнего коммита из GitHub.
